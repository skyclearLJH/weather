# 예상 강수량 방송 그래픽 자동 생성 — 스펙 축적 문서

통보문(예보 텍스트) + 완성 그래픽 쌍을 보고 추출한 규칙 모음.
사용자가 세트를 추가로 제공할 때마다 갱신한다.

## 제작 조건 (사용자 지시, 세트4에서 갱신)

- 소나기에 의한 예상 강수량 블록은 **모두 제외**
- **통보문 어딘가에 최대치(많은 곳 포함) 100mm 이상이 있으면 제작 대상 통보문**
- 제작 대상 통보문이면 **모든 예상 강수량 날짜 블록을 각각 1장씩 제작**
  - 세트4: 10일(120↑) + 11~12일(최대 80) → **2장** (사용자 확정: 100 미만 블록도 제작)
  - 미미한 블록 생략 기준(사용자 확정, 세트9 질의 후):
    1. 블록 최대값(많은 곳 포함)이 **20mm 미만**이면 생략 (권역 수 무관)
    2. **권역이 1개뿐**이고 블록 최대값이 **100mm 미만**이면 생략
       (세트9의 '11일 제주 5~40(60↑)' 단일 블록 → 생략, 사용자 확인)
    3. 그 외에는 모두 제작 (세트4 11~12일: 3개 권역·최대 80 → 제작)
- 날짜 범위 블록: `* 예상 강수량(11~12일)` → 한 장으로 제작 (사용자 확정)
  - 좌측 기간 표기: 상대 날짜 "(내일~모레, mm)"
  - **오늘부터 시작하는 범위는 "~"로 축약**: 16~17일(오늘~내일) → "(~내일, mm)" (세트8 확인)
  - 블록 안의 특정일 한정 항목 `(11일) 대구.경북남부: 5~40mm` → 해당 라벨 위에
    작은 글씨 "(내일)" 식 상대 날짜 표기

## 레이아웃

- 16:9 전체화면. 배경: 비 관련 사진(어두운 톤), 좌측에 사선 어두운 패널
- 좌측 타이틀: "예상 강수량" 큰 흰 글씨 + 밑줄 + 아래 작은 글씨 "(오늘, mm)" — 기간 라벨은 대상일에 따라 (오늘/내일 등)
- 지도: 화면 중앙. 남한 시도(예보구역) 폴리곤, 흰 외곽선, 회색 그림자
- 기자가 주로 화면 오른쪽에 서는 구도를 고려해 우측 약 20~25%를 진행자 안전 영역으로 비우고,
  좌측 제목 패널은 지도와 겹치지 않는 범위에서 좁게 사용
- 울릉도는 경계 데이터의 실제 지형만 사용하고 별도의 보정 원을 올리지 않음. 독도는 실제 동쪽 방향에 더 작은 점으로 보정 표시
- 울릉도·독도 강수량 라벨은 두 섬을 가리지 않도록 서쪽 또는 인접 바다에 배치
- 서해5도: 서해상 작은 섬 폴리곤은 표시하되 **"서해5도" 텍스트는 표시하지 않음**
- 제주: 남쪽에 별도 폴리곤 + 라벨

## 구역 색칠 규칙

- 예보구역(통보문의 구역 단위)별로 단색 채움
- **강수 예보가 없는 권역(소나기만 있는 권역 포함)은 회색**
- "많은 곳 [세부지역] Nmm 이상"이 특정 세부지역에 한정되면:
  그 세부지역을 **한 단계 더 진한 색의 별도 구역**으로 칠하고 라벨은 "N↑"만 단독 표기.
  본체 범위 라벨에는 많은 곳 값을 합치지 않음. 예: 경기남부만 `200↑`, 그 외 수도권은 `50~150`.
  (세트2: 강원중.북부내륙 120↑ 진남색, 충남북부 80↑ 진파랑)
- 시도 내부가 갈리는 구역(강원내륙.산지/동해안, 충남북부/남부, 충북중.북부/남부,
  경기남부 등)은 시군 조합 매핑으로 처리해야 함
- 세부 구역은 기상청 공식 예보용어의 시·군·읍면동 정의를 기준으로 매핑하고, 광역도 안에서
  하나의 강수 구역으로 묶인 시·군 사이의 내부 경계선은 표시하지 않음
- 경북처럼 시군 조합이 많은 구역도 각 폴리곤의 미세한 틈이 보이지 않도록 같은 채움색 선을
  겹쳐 그려 하나의 면처럼 표현

## 색상 단계 (관찰値, 상한 또는 '많은 곳' 값 기준으로 진해짐)

| 대표값(최대 기준) | 색 계열 | 관찰 사례 |
|---|---|---|
| ~5(내외) | 거의 흰 하늘 | 5안팎 (세트3 제주·강원동해안) |
| ~20 | 아주 연한 하늘 | 5~20 (세트1·2) |
| ~40 | 연하늘 | 5~40 (세트1·2·3) |
| ~50-70 | 중간 파랑 | 10~50/10~60/10~70 (세트3) |
| ~60-80 | 파랑(로열블루) | 10~60(세트1), 20~80(세트2·3) |
| ~80(많은곳) | 진파랑 | 80↑ (세트2 충남북부) |
| ~100 | 남색 | 30~80(100↑) 세트3 전라 |
| ~120 | 남색(짙은 네이비) | 30~80(120↑) 세트1·2 |
| ~150 | 보라 | 50~100(150↑) 세트1, 세트2 수도권 |
| ~250 | 진보라(거의 검보라) | 50~150(250↑) 세트1 |

- 같은 값이라도 그날 이벤트의 최대 단계에 따라 상대적으로 배색되는 경향
  (세트3처럼 최대 100인 날은 전체가 파랑 계열, 세트1처럼 250인 날은 보라까지 사용)
- **상대 배색으로 확정(사용자 결정)**: 색 서열(연하늘→하늘→파랑→남색→보라→자주→검보라)은
  고정, 그날 값 단계들을 낮은 값부터 서열에 순서대로 배치. 단계 수가 적으면 서열의
  아래쪽(파랑 계열)만 사용, 많거나 극값이 크면 자주·검보라까지 확장

## 라벨 규칙

- 기본 범위는 `최소~최대`, 세부지역에 한정된 많은 곳 값은 해당 진한 구역에 `N↑`로 분리 표기
  - 예: 수도권 본체 `50~150`, 경기남부 `200↑`
  - 서로 인접해도 값이 다른 `200↑`과 `150↑` 구역·라벨은 합치지 않음
- 같은 강수량 범위는 지도 전체에 라벨을 한 번만 표시. 같은 색 구역이 여러 곳이어도 숫자를 반복하지 않음
- `Nmm 내외` → 라벨 `N안팎` (세트3 제주 `5안팎`)
- 라벨 박스 배경 = 해당 구역 색, 흰색 굵은 숫자
- 라벨 숫자 크기와 비율은 디자이너 최신 결과를 기준으로 하고, 박스의 좌우·상하 여백은 숫자에
  타이트하게 맞춤
- 수치 박스와 숫자는 기존 자동 생성본 대비 약 70% 크기로 사용
- 기간 예외가 있는 구역은 라벨 위에 작은 흰 글씨로 `(~내일 아침)` 식 표기
  (통보문의 "(수도권, 20일 아침까지)" → "(~내일 아침)"으로 상대화)
  - 세트3: 수도권·강원내륙산지 두 구역에 각각 `(~내일 아침)` 부착 확인
  - **대상일을 넘겨 연장되는 기간만 표기**: `강원동해안(19일 낮까지)`(당일 내 종료),
    `충남북부 20일 새벽까지` 등은 세트1 그림에서 생략됨 (세트1 재검토로 확인)
  - `(내일부터)` 같은 시작일 주석은 기존 위치를 유지하되 수치보다 확실히 작은 크기로 표기
- 작은 섬(제주 등)은 '많은 곳 산지 100mm 이상' 같은 세부지역 한정이 있어도
  별도 구역으로 나누지 않고 통합 라벨 1개: `10~60(100↑)`
  - 세트7: 제주가 3개 항목(산지.중산간 30~80(100↑) / 그 외 20~60 / 북부 5~30)인데
    그림은 `20~60(100↑)` 라벨 1개 — 본체(가장 넓은 항목) 범위 + 최대 많은곳 값으로 통합
- **같은 '많은 곳' 값의 인접 구역은 하나의 진한 구역·라벨로 묶임** (세트7:
  경기남부 180↑ + 대전.세종.충남 180↑ → 연결된 진자주 구역에 `180↑` 라벨 1개;
  강원중.남부내륙 150↑ + 전북서부 150↑ 계열도 유사) ← 채택 여부 사용자 확인 권장
- 구형 템플릿(세트7, 크롭 이미지): 라벨 박스에 구역을 가리키는 삼각형 포인터 사용,
  배경 밝은 회색 — 목표는 최신 전체화면 템플릿이므로 삼각형은 미채택
- **세트1 그림의 제주 라벨 `10~60(10↑)`는 디자이너 오타** (통보문: 많은 곳 산지 100mm 이상)
  — 자동화 시 `10~60(100↑)`가 정답
- 라벨은 **해당 구역 위에 배치하는 것이 기본**이며, 구역 중심에 고정하지 않고 가장 읽기 좋은 내부 공간을 사용
- 제주나 경기남서부처럼 구역이 작거나 라벨끼리 밀집한 경우에만 해당 면을 완전히 가리지 않도록 대부분을
  바깥으로 빼고, 라벨 가장자리만 구역 일부와 겹치게 배치
- 모든 구역에 라벨을 달지는 않음(작은 값·병합 구역은 생략되기도) — 단 이는 디자이너 재량,
  구현은 통보문 구역별로 라벨을 만들되 배치 단계에서 조정

### 라벨 배치 우선순위 (18~19일 그래픽 피드백으로 확정)

1. **다른 강수 구역을 덮지 않는 것이 최우선**이다. 라벨 박스가 값·색이 다른 구역 위에 놓여
   그 구역의 라벨처럼 읽히는 배치는 금지한다. 예를 들어 `80~150`을 `250↑` 구역 위에 놓거나,
   `150↑`을 `300↑`·`100~200` 구역 위에 걸쳐 놓지 않는다.
2. **기본 위치는 대상 구역 위**다. 라벨이 들어갈 공간이 충분하면 해당 색상 면의 읽기 좋은 내부에
   올린다. 단, 라벨 중심을 대상 구역 중심에 정확히 맞출 필요는 없으며 두 중심 사이의 거리는
   낮은 우선순위의 참고값으로만 사용한다.
3. 바다 쪽 배치는 기본값이 아니라 **예외 규칙**이다. 대상 구역이 작거나, 다른 값 구역을 침범하거나,
   주변 라벨과 너무 가까워지는 경우에만 가까운 바다 쪽으로 이동한다. 이때 라벨의 모서리나
   가장자리만 대상 구역에 걸치게 하고, 박스 대부분은 바다에 두어 대상 면과 인접 구역을 모두
   읽을 수 있게 한다.
4. 후보 위치는 대상 구역 안의 빈 공간, 대상 구역과 맞닿은 같은 값 구역, 인접 바다 순으로 탐색한다.
   값이 다른 이웃 구역 위에 올리는 후보는 사용하지 않는다.
5. 배치 순서는 **면적이 작고 후보 공간이 적은 세부 구역(많은 곳 포함)부터**, 넓은 본체 구역
   순으로 한다. 먼저 놓인 라벨과 겹치면 후순위 라벨을 바다나 외곽으로 이동한다.
6. 자동 배치의 충돌 비용은 `다른 값 구역 침범 > 다른 라벨과 겹침 > 대상 구역을 과도하게 가림
   > 대상 구역 중심에서 멀어짐` 순으로 둔다. 즉 중심에서 벗어나더라도 오표현과 겹침을 피한다.
7. 너무 가까워 별도 박스를 놓을 수 없고 **동일한 강수량 문구가 양쪽 구역에 모두 정확히 적용될
   때만** 하나의 라벨로 병합할 수 있다. 병합 라벨의 대상은 두 구역의 합집합이며, 두 구역 사이
   또는 함께 읽히는 바다 쪽에 둔다.
8. 서로 다른 범위나 다른 극값(`200↑`과 `150↑` 등)은 가까워도 병합하지 않는다. 범위가 일부
   겹친다는 이유로 공통 숫자만 뽑아 새 라벨을 만드는 것도 금지한다.

## 통보문 파싱 패턴 (관찰된 변형)

- 블록: `* 예상 강수량(N일)` ~ 다음 `*` 또는 끝. `* 소나기에 의한 …`은 제외
- 줄 형식: `- (권역) 구역[, 구역]: 범위mm(많은 곳 [세부지역] Nmm 이상)/ 구역: …`
- 기간 한정 변형:
  - 권역 옆: `- (수도권, 20일 아침까지) …`
  - 구역 옆: `강원동해안(19일 낮까지): 5~20mm`
  - 권역 옆 일부 구역만: `- (충청권, 충남북부 20일 새벽까지) …`
  - **날짜 범위 블록 안에서 권역 옆 단일 날짜**: `(수도권, 9일) 경기남부: 5mm 안팎`
    → 그 권역 항목 전체가 해당 날짜에만 적용 (라벨 위 "(오늘)" 식 표기 대상, 세트9)
  - **블록 날짜와 다른 날을 가리키는 권역 한정**: 9일 블록 안 `(제주도, 10일) 제주도: 5~10mm`
    → "(내일)" 표기 대상 (세트10)
  - `(수도권, 10일까지)`(=내일까지) → 세트10 그림은 "~내일 오전"으로 표기 — 본문 예보의
    그침 시각(오전)을 디자이너가 반영해 구체화한 것. **자동화는 통보문 블록 정보만으로
    "(~내일)"까지 표기**하고 본문 참조 구체화는 하지 않음(안전 우선)
- 최대값 변형: `많은 곳 120mm 이상` / `많은 곳 강원남부내륙 100mm 이상`(세부지역 한정)
  / `많은 곳 전남해안(북부서해안 제외) 120mm 이상`(세부지역에 다시 괄호 제외)
  / **다중**: `많은 곳 전북 120mm 이상, 광주.전남북부 100mm 이상` — 값이 다른 세부지역 2개,
    각각 별도의 진한 구역으로 칠하고 라벨 분리(세트5: 전북 `120↑` 자주 단독,
    전남 본체 `30~80(100↑)` 보라)
  / 세트9: `광주.전남: 50~100(많은 곳 전남해안 200↑, 광주.전남내륙 150↑)` →
    본체 남색 + 내륙 150↑ 보라 + 해안 200↑ 진자주의 **3단 중첩 배색** 확인
    (인접 경상권의 같은 값 많은곳(200/150)과 각각 묶여 표현)
  / 같은 값 나열형: `많은 곳 전북, 전남북서부 200mm 이상` — 세부지역 2곳이 한 값 공유 (세트10)
- 구역명 세분 추가 수집(세트10): `대전.세종.충남(북서부 제외)`, `광주.전남북서부.중부서해안`,
  `전남동부.남서부(중부서해안 제외)`, `경북남서내륙`, `경북남동부`, `경남서부내륙`, `충남북서부`
- 소량 표현: `5mm 내외` / `5mm 미만` / `5mm 안팎` 모두 등장 — 그림에서는 일괄 `5안팎` 표기
  (세트5: 경기내륙 '5mm 미만' → 라벨 '5안팎')
- 제외 표현: `광주.전남(남서부 제외)` / `울산.경남내륙(경남남서내륙 제외)`
  — 구역명 뒤 괄호로 세부지역 제외, 제외된 세부지역은 대개 별도 항목으로 등장
- 복합 구역명: `부산.경남남해안.경남남서내륙`처럼 점(.)으로 3개 이상 연결
- 많은곳 세부지역 명칭 다양성: `경기남부`, `전북서부`, `전남북부서해안`, `지리산부근`(!),
  `충남남동부`, `강원중.남부내륙`, `부산.경남남해안.지리산부근`(복합)
  — 시군 경계와 무관한 지형 명칭(지리산부근)도 등장 → 매핑 불가 시 본체 라벨에만 반영
- 제주 세분: `제주도산지.중산간` / `제주도(북부.산지.중산간 제외)` / `제주도북부`
- 서해5도가 본체 항목에 포함되기도: `서울.인천.경기, 서해5도: 50~150mm(…)` (세트8)
- 울릉도.독도가 대구.경북 항목에 묶이기도: `대구.경북(…), 울릉도.독도: 10~60mm` (세트3·8)

## 구현 방식 결정 (사용자 확정)

- 색상: 상대 배색
- 진입: 방송모드처럼 숨김 버튼(URL 게이트), 구체 UI는 결과물 보고 사용자가 결정
- 날짜 범위 블록: `* 예상 강수량(11~12일)`; 블록 안에 특정일 한정 항목 가능:
  `(11일) 대구.경북남부: 5~40mm`
- 빗방울: `0.1mm 미만의 빗방울` — 예상 강수량 아님(본문에만 등장)
- 배경은 흐릿한 그래픽 패턴이 아니라 실제 비가 내리는 장면의 사진을 사용

## 디자이너 임의 단순화 (구현에 반영하지 말 것 — 사용자 확인 완료)

- 위 라벨 배치 규칙의 **동일 문구·인접 구역 병합**만 예외적으로 허용한다. 서로 다른 범위를
  합집합 범위로 바꾸는 아래 사례들은 계속 자동 구현에 반영하지 않는다.

- 색 단계가 너무 많으면 비슷한 값·인접 구역을 병합하고 범위는 합집합으로 표기
  - 세트2: 충북중.북부 20~60 + 강원내륙.산지 30~80 → 20~80
  - 세트2: 강원북부동해안 10~40 → 5~40 색 구역에 병합(라벨 생략)
  - 세트3: 대구.경북 10~60 + 부산.울산.경남 20~70 → "10~70" 단일 라벨
  - 세트3: 충남서해안 20~80은 수도권 20~80 색과 동일화, 충청내륙 10~60은 경상과 병합
- 라벨 축약: 세트2 수도권 50~120(150↑) → "50~150↑", 세트3 전라 30~80(100↑) → "100↑"
  ← 구현은 항상 원칙 형식 `최소~최대(많은곳↑)` 사용 (사용자 확정)
- 세트4: 많은곳 세부지역(전남해안·경남남해안 120↑)을 보라 별도 구역으로 칠하면서
  라벨은 통합형 "30~100(120↑)" 하나만 보라 배경으로 표기 (30~100 본체는 남색)
  → 구현 종합: 본체 구역 = 상한 기준색, 많은곳 세부지역 = 한 단계 진한 색,
    라벨은 원칙 형식 1개(필요 시 세부지역에 N↑ 별도)

## 구현 방향 (합의)

- A안: 통보문 자동 파싱 → 초안 그래픽 자동 생성 → 사람이 라벨 위치 등 미세조정(반자동)
- 세부 구역(내륙/산지/동해안/북부/남부) → 시군 조합 매핑 테이블 필요
- 기존 방송모드 지도 인프라(MapLibre + admdongkor 경계) 재활용 가능
